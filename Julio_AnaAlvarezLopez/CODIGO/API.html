Descripción General

Este sistema proporciona un marco completo para verificar y probar la estructura, formato y envío de formularios para entidades de base de datos. Está compuesto por cuatro clases principales que trabajan juntas para validar datos y ejecutar pruebas.

Arquitectura del Sistema

┌─────────────────────────────────────────┐ │ ENTRY POINT: index.html │ │ (Menu → Gestor → TestForm/TestSubmit) │ └─────────────┬───────────────────────────┘ │ ├──→ Gestor_Class.js │ └─ Carga entidades y validación de estructura │ ├──→ ValidateFieldsForm_Class.js │ └─ Crea campos HTML y ejecuta validaciones │ ├──→ Validations_Class.js │ └─ Métodos estáticos de validación │ ├──→ TestForm_Class.js │ └─ Verifica estructura, definiciones y datos │ └──→ TestSubmit_Class.js └─ Prueba validación de envío por acción

Flujo de Uso Típico

  1. Usuario selecciona entidad del menú en index.html
  2. Gestor carga estructura y tests de la entidad
  3. Usuario puede elegir:
    • TestForm: Verifica estructura de atributos y ejecuta pruebas unitarias
    • TestSubmit: Verifica validación de formularios por acción (CREATE, UPDATE, etc.)
  4. Cada clase muestra resultados en modal con opción de ver detalles

Validations_Class

Proporciona métodos estáticos para realizar validaciones individuales en campos de formulario. Cada método recibe el ID del elemento HTML y retorna true o un mensaje de error.

Descripción

Clase con métodos estáticos para validar campos individuales. Todos los métodos trabajan con IDs de elementos HTML del DOM.

Métodos

Static isRequired()
Validations.isRequired(elementId) → boolean|string
Valida que el campo no esté vacío
Parámetros:
Nombre Tipo Descripción
elementId string ID del elemento HTML a validar
Retorna: true si es válido, o string con mensaje de error
Static isInteger()
Validations.isInteger(elementId) → boolean|string
Valida que el valor sea un número entero
Retorna: true si es válido
Static hasMinSize()
Validations.hasMinSize(elementId, minSize) → boolean|string
Valida que el valor tenga un tamaño mínimo
Parámetros:
Nombre Tipo Descripción
elementId string ID del elemento HTML
minSize number Tamaño mínimo requerido
Static hasMaxSize()
Validations.hasMaxSize(elementId, maxSize) → boolean|string
Valida que el valor tenga un tamaño máximo
Parámetros:
Nombre Tipo Descripción
elementId string ID del elemento HTML
maxSize number Tamaño máximo permitido
Static noAccents()
Validations.noAccents(elementId) → boolean|string
Valida que el valor no contenga acentos
Static noSpaces()
Validations.noSpaces(elementId) → boolean|string
Valida que el valor no contenga espacios
Static isValidDate()
Validations.isValidDate(elementId) → boolean|string
Valida que el valor sea una fecha válida (formato DD-MM-YYYY)
Static isValidFileType()
Validations.isValidFileType(elementId, validTypes) → boolean|string
Valida que el archivo sea de un tipo permitido
Static isValidOption()
Validations.isValidOption(elementId, optionValues) → boolean|string
Valida que el valor sea una opción válida de un select

ValidateFieldsForm_Class

Crea elementos HTML para campos individuales basados en la definición de estructura de la entidad. Integra con Validations_Class para ejecutar validaciones.

Constructor

new ValidateFieldsForm(fieldDefinition, value = '')
Parámetro Tipo Descripción
fieldDefinition Object Objeto con propiedades: name, type, validations, etc.
value string Valor inicial del campo (opcional)

Métodos

Instance createField()
createField() → string
Genera el HTML del campo selon su tipo (input, textarea, select, date, file)
Retorna: HTML del campo como string
Instance getValue()
getValue() → *
Obtiene el valor actual del campo desde el DOM
Instance setValue()
setValue(value)
Establece el valor del campo en el DOM
Instance validateField()
validateField(entityClass = null) → Object
Ejecuta todas las validaciones del campo
Retorna: Objeto con estructura:
{ fieldName: string, isValid: boolean, errors: Array, warnings: Array }
Instance clear()
clear()
Limpia el valor del campo

TestForm_Class

Verifica la integridad de la estructura de la entidad, las definiciones de pruebas y los datos de prueba. Ejecuta todas las validaciones y muestra resultados detallados.

Constructor

new TestForm(entityName, structure)
Parámetro Tipo Descripción
entityName string Nombre de la entidad a probar
structure Object Objeto de estructura de la entidad con propiedades attributes, etc.

Métodos Principales

Instance analyze()
analyze() → boolean
Inicia el análisis completo: estructura, definiciones, datos y ejecución
Retorna: true si tuvo éxito
Instance analyzeStructure()
analyzeStructure() → Object
Valida la estructura de la entidad
Retorna: Objeto con: {valid, count, attributes, attributeDetails}
Instance analyzeTestDefinitions()
analyzeTestDefinitions() → Object
Analiza las definiciones de pruebas
Retorna: {count, wellDefined, issues}
Instance analyzeTestData()
analyzeTestData() → Object
Analiza los datos de prueba
Retorna: {count, wellTyped, byType, issues}
Instance executeAllTests()
executeAllTests() → Object
Ejecuta todas las pruebas definidas
Retorna: {executed, correct, incorrect, results}
Instance displayResults()
displayResults(analysisResults)
Muestra los resultados en una ventana modal

TestSubmit_Class

Verifica la validación de formularios para todas las acciones (CREATE, UPDATE, EDIT, DELETE, READ). Ejecuta CheckSubmit() y valida que el comportamiento sea el esperado.

Constructor

new TestSubmit(entityName, testSubmitData)
Parámetro Tipo Descripción
entityName string Nombre de la entidad
testSubmitData Array Array de pruebas de formulario con estructura: {action, formData, expectError, description}

Métodos Principales

Instance analyze()
analyze() → boolean
Inicia el análisis de todas las acciones
Retorna: true si tuvo éxito
Instance analyzeAllActions()
analyzeAllActions() → Object
Analiza CREATE, UPDATE, EDIT, DELETE, READ
Retorna: Objeto con análisis para cada acción
Instance executeSubmitTest()
executeSubmitTest(action, test, index) → Object
Ejecuta una prueba individual de envío
Parámetro Tipo Descripción
action string CREATE|UPDATE|EDIT|DELETE|READ
test Object Objeto de prueba a ejecutar
index number Índice de la prueba
Retorna: {testNumber, action, expectError, hasErrors, correct}
Instance displayResults()
displayResults(analyses)
Muestra resumen en modal con opción de ver detalles

Gestor_Class

Administra la carga de entidades y coordina la ejecución de TestForm y TestSubmit. Valida la existencia de archivos y estructura de la entidad.

Constructor

new Gestor(entityName)
Parámetro Tipo Descripción
entityName string Nombre de la entidad a gestionar

Métodos Principales

Instance loadEntity()
loadEntity() → boolean
Carga la estructura, tests y TestSubmit de la entidad. Valida la existencia de: nombreentidad_estructura, nombreentidad_tests, nombreentidad_TestSubmit
Retorna: true si tuvo éxito
Instance validateStructure()
validateStructure() → boolean
Valida que la estructura existe y es válida
Instance checkVariablesExistence()
checkVariablesExistence() → Object
Verifica la existencia de todas las variables requeridas
Retorna: {missing, errors}
Instance showTestForm()
showTestForm()
Crea y muestra una instancia de TestForm
Instance showTestSubmit()
showTestSubmit()
Crea y muestra una instancia de TestSubmit

Workflows y Diagramas de Flujo

Workflow 1: TestForm - Verificación de Estructura y Ejecución de Pruebas

User selecciona entity ↓ Gestor.loadEntity() ↓ Valida estructura ↓ User hace clic en "Verificar Tests" ↓ new TestForm(name, structure) ↓ testForm.analyze() ├─→ analyzeStructure() │ └─→ Verifica attributes │ ├─→ analyzeTestDefinitions() │ └─→ Carga def_tests_entity │ ├─→ analyzeTestData() │ └─→ Carga tests_fields_entity │ └─→ executeAllTests() └─→ Para cada test: ├─ Crea campo temporal ├─ Ejecuta validación └─ Compara resultado esperado ↓ displayResults() ├─ Modal con resumen └─ Botón "Ver Detalles" ↓ showDetailedResults() └─ Tabla completa de tests

Workflow 2: TestSubmit - Verificación de Envío de Formulario

User selecciona entity ↓ Gestor.loadEntity() ↓ User hace clic en "Verificar Submit" ↓ new TestSubmit(name, testSubmitData) ↓ testSubmit.analyze() ↓ analyzeAllActions() ├─→ Para CREATE: │ └─→ Valida todos los campos requeridos │ ├─→ Para UPDATE/EDIT: │ └─→ Valida ID + campos actualizables │ ├─→ Para DELETE: │ └─→ Valida solo ID │ ├─→ Para READ: │ └─→ Valida ID o criterio de búsqueda │ └─→ Para SEARCH: └─→ Valida criterios de búsqueda ↓ displayResults() ├─ Modal con resumen por acción └─ Botón "Ver Detalles" ↓ showDetailedResults() └─ Tabla completa por acción

Datos de estructura esperados

La estructura de la entidad debe contener:

nombreentidad_estructura = { attributes: [ { name: 'id_entity', type: 'input', validations: [ { rule: 'required' }, { rule: 'minSize', params: { size: 1 } }, { rule: 'maxSize', params: { size: 10 } } ] }, // ... más atributos ] };

Datos de pruebas esperados

Las pruebas deben contener:

def_tests_nombreentidad = [ { id: 1, name: 'Test 1', rule: 'required', params: {} }, { id: 2, name: 'Test 2', rule: 'minSize', params: { size: 1 } } // ... ]; tests_fields_nombreentidad = [ { id: 1, value: '', error: true }, // Espera error { id: 2, value: 'a', error: false }, // Espera éxito // ... ];

Datos de submit esperados

Los tests de submit deben contener:

nombreentidad_TestSubmit = [ { action: 'CREATE', description: 'Crear entidad válida', formData: { id: '1', name: 'test' }, expectError: false }, { action: 'UPDATE', description: 'Actualizar sin ID', formData: { name: 'test' }, expectError: true } // ... ];
